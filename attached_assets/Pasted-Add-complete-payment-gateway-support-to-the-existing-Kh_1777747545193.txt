Add complete payment gateway support to the existing Khidma backend and frontend.

Do not rebuild the project.
Keep the current Drizzle/Express backend and existing frontend architecture.

Payment gateway scope:
1. Stripe
2. PayTabs
3. Telr
4. Manual bank transfer
5. Cash or offline admin-confirmed payment

Main requirement:
Create a payment gateway abstraction layer so the platform can support multiple payment providers without changing the contract, milestone, escrow, invoice, or payout logic.

Backend requirements:
- Create a unified PaymentGateway interface
- Implement Stripe adapter
- Implement PayTabs adapter placeholder with real request structure
- Implement Telr adapter placeholder with real request structure
- Implement manual payment adapter
- Add gateway configuration through environment variables
- Add platform setting for active gateway
- Allow admin to enable/disable gateways

Payment workflows:
1. Client funds contract or milestone
2. Backend creates payment intent/session with selected gateway
3. Payment provider redirects or confirms payment
4. Webhook/callback updates payment status
5. Funds move into escrow-held state
6. Freelancer submits work
7. Client approves milestone
8. Escrow release creates wallet transaction
9. Admin processes payout manually or through future payout API
10. Invoice is generated after successful payment

Manual bank transfer flow:
1. Client selects bank transfer
2. System generates pending transaction
3. Client sees bank account instructions
4. Client uploads transfer proof
5. Admin reviews proof
6. Admin approves or rejects
7. If approved, transaction becomes PAID and escrow becomes HELD
8. If rejected, transaction becomes FAILED with rejection reason

Stripe flow:
1. Client selects Stripe
2. Backend creates Stripe Checkout Session or PaymentIntent
3. Frontend redirects user or confirms payment
4. Stripe webhook confirms payment
5. Backend marks transaction as PAID
6. Backend creates escrow ledger entry
7. Backend generates invoice
