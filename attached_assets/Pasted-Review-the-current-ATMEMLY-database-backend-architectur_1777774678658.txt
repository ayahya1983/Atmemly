Check and improve:

1. Database best practices
- Proper relational structure
- No unnecessary duplicated data
- Correct foreign keys
- Proper indexes on frequently queried fields
- Soft delete where needed
- createdAt/updatedAt timestamps on all major tables
- Audit logs for sensitive actions
- Transaction-safe payment, escrow, payout, and wallet logic
- No frontend-controlled financial state changes
- Migration safety using Drizzle
- Seed data separated from production data
- Clear enum/status fields for workflows

2. Scalability
- Pagination on all list APIs
- Filtering and sorting on admin tables
- Search-ready indexes
- Avoid loading large datasets at once
- Avoid N+1 query patterns
- Use database transactions for multi-step workflows
- Keep services modular
- Prepare cache layer only where useful
- Prepare file storage abstraction for S3/R2
- Keep payment gateway abstraction modular

3. Backend best practices
- Modular route/service/controller structure
- Zod validation on request bodies and query params
- Centralized error handling
- Role-based and permission-based middleware
- Rate limiting for auth, payments, and admin actions
- Helmet/CORS security configuration
- Structured logging
- Health check endpoint
- Environment variable validation
- Idempotency keys for payments and webhooks
- Audit logging for admin and financial changes

4. Frontend best practices
- Separate marketplace and admin apps
- Shared API client where possible
- Shared types where possible
- No hardcoded production data
- Loading, empty, and error states
- Protected routes
- Role-based admin UI visibility
- RTL-first Arabic layout
- Component reuse
- Responsive design
- Clean state management

5. Financial data integrity
- Wallet balance must be derived from ledger or transaction-safe updates
- Escrow release must not happen twice
- Refund must not happen after release
- Payment status must only change through backend, webhook, or admin-secured action
- Webhook processing must be idempotent
- Payout cannot exceed available freelancer balance
- Invoices must reference immutable payment snapshots

6. Admin scalability
- Server-side pagination
- Server-side filters
- Server-side search
- CSV export via backend
- Audit logs for every mutation
- Permission checks for every action
